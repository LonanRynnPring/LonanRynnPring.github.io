Codenation



People who are blind or who have sight issues may struggle to read any written content of a webpage, to overcome this we can implement a text to speech function that would read out the written content of a page and describe any images or other visual content. Most devices have a text-to-speech function, so we would just have to write a script to access it for our webpage. Also to help with sight issues and general comprehension it is important to choose a good font and a large font size. The colour of the font and the background is important too, if the colours clash it would be harder to read.

HTML Accessibility

It is good practice to write your HTML code with accessibility in mind. Your aim is to provide the user a good way to navigate and utilise your webpage. Semantic HTML is the practice of using correct HTML elements for their correct purpose as much as you can. Semantic elements are elements with a meaning, e.g. using the button element instead of a div element for a button. This give context to a screen reader and thus the webpage is more accessible to people using a screen reader. Semantic HTML is also accessible for people using keyboard only navigation, it can be accessed using the tab key.

Screen readers also read the alt attribute for images, so if you describe what the image is in the alt attribute people using a screen reader will know what it shows. Use language that is easy to understand and easily readable by a screen reader, for example avoid using dashes, instead of writing 1-5, write 1 to 5.[2]



Other



There are lots of file types you can use for websites. For images there are, PNG JPG GIF and SVG. They all serve different purposes.A JPG is good for real life images and they are compressed which means it’ll be a small file. GIFs are good for animation, they only accommodate up to 256 index colours and as a result are very small files, they are good for adding motion to your website. PNGs can be animated like GIFs and they can support up to 16 million colours like JPGs, as a result they are much more detailed and are often bigger files. They also support transparency. SVGs are drawn mathematically and not with pixels unlike the others, they tend to be larger files than the other types but are also the most detailed. HTML is used for the content of a webpage, splitting the page into Headings, paragraphs of text and images. CSS is used for styling the HTML however you want, giving it colour, borders, layout, etc.[3]
